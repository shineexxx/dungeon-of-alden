# AGENTS.md — Инструкции для ИИ-ассистентов

Файл предназначен для ИИ-агентов, работающих с проектом. Предполагается, что читатель ничего не знает об этой кодовой базе. Вся основная документация и комментарии в проекте написаны на русском языке — новые материалы, комментарии и сообщения пользователю следует писать на русском.

## Обзор проекта

**Название:** Подземелье Альдена  
**Тип:** консольный roguelike с процедурной генерацией подземелий.  
**Цель игры:** спуститься на 7-й уровень (финальный тронный зал), победить босса — Тёмного Владыка — и попасть в Зал славы.

Игра реализована на **Python 3.12** с использованием стандартной библиотеки `curses` для отрисовки интерфейса в терминале. Внешних зависимостей нет. Проект не использует `pyproject.toml`, `setup.py`, `requirements.txt` или аналогичные файлы пакетирования — это самодостаточное приложение, запускаемое напрямую через `python3 game.py`.

Ключевые механики:
- процедурная генерация уровней и биомов (каменное подземелье, пещеры, склеп, грибной лес, кузница ада, тёмные глубины);
- пошаговый бой, магия с целевыми типами (self/single/line/cone/aoe), инвентарь, экипировка, голод, ловушки, NPC, спецкомнаты, журнал (бестиарий/лор), сохранения/загрузки и Зал славы.

## Технологический стек и архитектура

- **Язык:** Python 3.12+.
- **Графический интерфейс:** стандартная библиотека `curses` (терминал, UTF-8).
- **Требования к терминалу:** поддержка UTF-8, минимальный размер окна **60×14**.
- **Сериализация:** JSON (сохранения, настройки, Зал славы).
- **Случайность:** `random.Random` с фиксированным `seed` для воспроизводимости.
- **Типизация:** везде используются аннотации типов и `from __future__ import annotations`.

### Ключевые сущности

- `systems.game_state.GameState` — центральный dataclass, содержащий всё состояние партии: игрока, мобов, подземелье, предметы на полу, журнал, RNG, настройки. Имеет `to_dict()` и `from_dict()` для сериализации.
- `entities.player.Player` — dataclass с характеристиками, инвентарём, экипировкой, заклинаниями и статус-эффектами.
- `entities.mob.Mob` — моб с AI, характеристиками и энергией (speed-based turns).
- `world.dungeon.Dungeon` — сетка тайлов, FOV, explored, комнаты, лестница, hazards, ловушки, интерактивные объекты.

## Сборка и запуск

### Запуск игры

```bash
python3 game.py
```

### Запуск тестов

Все тесты — headless, не требуют настоящего терминала. Запускать по одному файлу:

```bash
for f in tests/test_*.py; do python3 "$f"; done
```

Проект не использует `pytest`/`unittest`. Тесты — это обычные Python-скрипты с функциями `test_*`, печатающие результат и завершающиеся успешно либо выбрасывающие исключение при ошибке.

### Проверка окружения

```bash
python3 --version   # ожидается 3.12+
python3 -c "import curses; print('curses ok')"
```

## Соглашения по стилю кода

- **Язык:** комментарии, докстринги, сообщения пользователю и `README.md` написаны на русском. Новый код должен сохранять этот стиль.
- **Импорты:** в начале каждого модуля используется `from __future__ import annotations`. Циклические импорты разрешаются через `TYPE_CHECKING` или локальные импорты внутри функций.
- **Типизация:** аннотации типов обязательны для публичных функций и классов. Для коллекций используются generics (`dict[str, dict]`, `list[Mob]` и т.п.).
- **Классы данных:** для сущностей состояния используются `dataclasses` (`@dataclass`) с сериализацией `to_dict()` / `from_dict()`.
- **Контент как данные:** большие таблицы предметов, монстров, заклинаний и биомов хранятся в виде словарей в `content/`. Каждая таблица содержит докстринг со схемой полей. Для добавления нового контента достаточно добавить новую запись в словарь, не трогая движок.
- **Эффекты:** в `systems/effects.py` зарегистрированы функции-эффекты по строковым ключам. Каждая функция возвращает строку сообщения для лога.
- **RNG:** весь случайный процесс идёт через `state.rng` (экземпляр `random.Random`), чтобы обеспечить детерминированность и корректную сериализацию.
- **Сохранения:** хранятся в `saves/*.json`. Текущая версия формата сохранений — **7** (`systems/save.SAVE_VERSION`). При изменении формата необходимо обновить `save_version` и добавить миграцию в `GameState.from_dict()` / `load_game()`.
- **Git:** директория `saves/` и `__pycache__/` исключены из git (см. `.gitignore`).

## Тестирование

- Тесты расположены в `tests/` и именуются `test_*.py`.
- Headless-режим достигается подменой модуля `curses` заглушкой (см. `tests/test_basic.py`).
- Каждый тестовый файл можно запускать самостоятельно: `python3 tests/test_*.py`.
- Перед коммитом следует прогонять весь набор тестов командой:

```bash
for f in tests/test_*.py; do python3 "$f" || exit 1; done
```

- При добавлении новой механики желательно добавлять новый `tests/test_<feature>.py` по образцу существующих.

## Инструкции по разработке

### Добавление нового контента

1. **Предмет:** добавить запись в `content/items.py`, указать `type`, `effect`, `rarity`, `depth_min`, `tags`.
2. **Оружие/броня:** добавить запись в `content/weapons.py` или `content/armor.py` и связать с `item_id` через `effect_power`.
3. **Монстр:** добавить запись в `content/monsters.py`, указать `ai`, `depth`, `tags`.
4. **Заклинание:** добавить запись в `content/spells.py` и соответствующую функцию-эффект в `systems/effects.py`, зарегистрировать в `EFFECTS`.
5. **Биом:** добавить запись в `content/biomes.py`, при необходимости реализовать генератор в `world/dungeon.py`.
6. **Событие/спецкомната:** добавить событие в `content/events.py` и обработчик в `systems/interactables.py`.

### Изменение игровой логики

- Все действия игрока обрабатываются в `engine/game_loop.run_game()`.
- Ходы мобов и голод обрабатываются в `_after_player_turn()` и `_apply_hunger()`.
- Бой игрока — `_player_attack()`, `_player_damage_range()`.
- AI мобов — `entities/ai.py`.
- Статус-эффекты — `systems/effects.py` (`apply_status`, `tick_status_effects`, `is_paralyzed`).

### Изменение формата сохранений

- Обновить `systems/save.SAVE_VERSION`.
- Обновить `GameState.to_dict()` и `GameState.from_dict()`.
- Добавить обработку старых версий в `systems/save.load_game()` или `GameState.from_dict()`.
- Проверить обратную совместимость тестом `tests/test_save_v6.py` (или аналогичным).

## Безопасность и надёжность

- **Сохранения — это JSON:** не сохраняйте исполняемый код, pickle или пользовательский ввод без валидации.
- **Путь `saves/`:** все файлы сохранений, настроек и Зала славы пишутся в эту директорию. Она исключена из git; не допускайте утечки сохранений пользователей в репозиторий.
- **Пользовательский ввод:** имена слотов сохранений обрезаются и декодируются с `errors="replace"`. Избегайте использования этих имён для построения путей за пределами `saves/`.
- **Curses:** игра зависит от терминала. Все вызовы `curses.curs_set()` обёрнуты в `try/except`, чтобы не падать в неподдерживающих эмуляторах.
- **Отсутствие сети:** игра оффлайн; не выполняйте сетевые запросы без явного требования.
